# Проект my_course_planes

## Описание:

Проект my_course_planes - это тестовое приложение для получения информации о самолетах в воздушных пространствах 
выбранных стран.

## Стек используемых технологий (библиотек):
+ Python 3.12.10
+ poetry
+ pytest
+ json
+ requests

## Установка:

1. Клонируйте репозиторий:
    ```
     https://github.com/ViktorPilot/planes.git
    ```
2. Запустите PyCharm у себя на компьютере.
3. Откройте скачанный репозиторий в PyCharm.
4. Создайте виртуальное окружение командой `poetry install`.
5. Активируйте ВО командой `poetry env activate`.
6. Установите зависимости командой `poetry init`.
7. Программа готова к работе.

## Запуск проекта через главный модуль main.py:

1. Необходимо запустить выполнение программы командой `poetry run python main.py`.
2. В Python файле `main.py` по запросу ввести:
    + название страны информацию о самолетах в которой необходимо получить.
    + диапазон высот полета самолетов через пробел.
    + количество самолетов для вывода в топ N по высоте полета.
    + названия стран для фильтрации по стране регистрации через запятую.

## Использование отдельных модулей:

### Для модуля api_interaction.py:

1. В Python файле `api_interaction.py` задать:
    + экземпляры класса `CoordsPlanes` в формате: `airplanes = CoordsPlanes()`.
    + к созданному экземпляру класса применить метод, возвращающий данные о самолетах в заданной стране, 
   задать название страны в формате: `airplanes.get_info_planes("Russia")`
2. Запустить выполнение программы командой `python -m src.api_interaction`.
3. В результате получим словарь с данными о самолетах в заданной стране при обращении к API.

### Для модуля plane_interaction.py:

1. В Python файле `plane_interaction.py` задать:
    + в методе для создания объектов класса `PlanesInfo`, полученных по API указать 
   необходимую страну: `PlanesInfo.create_obj("Russia")`.
    + два и более экземпляра класса `PlanesInfo` в формате: `airplane = PlanesInfo("Canada", "ACA411", 2200, 500)`.
    + для сравнения самолетов по высоте полета задать неравенство в формате: `airplane < airplane1`.
    + для сравнения самолетов по скорости полета задать неравенство в формате: `airplane > airplane1`.
2. Запустить выполнение программы командой `python -m src.plane_interaction`.
3. В результате получим список объектов с данными о самолетах в заданной стране при обращении к API и булево значение 
подтверждающее или опровергающее неравенство по высоте и скорости.
4. При этом проводится валидация правильности данных.

### Для модуля file_interaction.py:

1. В Python файле `file_interaction.py` необходимо задать:
    + экземпляры класса `WriteFileJson` в формате: `ap = WriteFileJson()`.
    + экземпляры класса `PlanesInfo` в формате: `airplane3 = PlanesInfo("Canada", "A1112", 2200, 500)`.
    + к созданным экземплярам класса `WriteFileJson` применить методы для чтения, добавления или 
   удаления информации о самолетах из json файла: 
   `ap.get_airplane()`
   `ap.add_airplane(airplane3)`
   `ap.del_airplane(airplane4)`.
2. Предусмотрена возможность изменить путь для работы с json файлом. 
Для этого при создании экземпляра класса `WriteFileJson` необходимо передать аргумент, 
представляющий собой новый адрес в формате: `../data/new_file.json`
3. Запустить выполнение программы командой `python -m src.file_interaction`.
4. В результате выполнится чтение, добавление или удаление информации о самолетах в json файле.

### Для модуля utils.py:

1. В Python файле `utils.py` необходимо задать:
   + экземпляры класса `WriteFileJson` в формате: `airplanes_country = WriteFileJson()`.
   + по запросу ввести диапазон высот полета через пробел в формате: `5000 9000`, 
   количество самолетов для вывода в топ N в формате: `3`,
   названия стран для фильтрации по стране регистрации через запятую(без пробела) в формате: `Canada,Russia`.
2. Запустить выполнение программы командой `python -m src.utils`.
3. В результате получим:
   + список самолетов, отфильтрованный по высоте полета, заданной пользователем.
   + топ N список самолетов, отсортированный по высоте полета.
   + список самолетов, отфильтрованный по странам регистрации, заданной пользователем.

## Тестирование:

Данный проект покрыт юнит-тестами с использованием фреймворка pytest.
Для их запуска выполните команду `poetry run pytest` из корневой директории проекта.

### Реализованы:

### Для модуля api_interaction.py:

+ Тестирование функции при возвращении от API статус кода `200`.
+ Тестирование функции при возвращении от API статус кода с ошибкой.
+ Тестирование функции, возвращающей json-ответ с данными по самолетам при успешном обращении к API.

### Для модуля plane_interaction.py:

+ Тестирование инициализации экземпляра класса `PlanesInfo`.
+ Тестирование классметода создания списка объектов основных данных по самолетам, полученных по API 
при валидных значениях.
+ Тестирование классметода создания списка объектов основных данных по самолетам, полученных по API при ошибках API.
+ Тестирование создания экземпляров класса `PlanesInfo` на валидность значений.
+ Тестирование метода сравнения самолетов по высоте полета.
+ Тестирование метода сравнения самолетов по скорости полета.

### Для модуля file_interaction.py:

+ Тестирование инициализации экземпляра класса `WriteFileJson`.
+ Тестирование сеттера пути к файлу.
+ Тестирование функции для чтения информации о самолетах из json файла при отсутствии файла по заданному адресу.
+ Тестирование функции для чтения информации о самолетах из json файла при валидных значениях.
+ Тестирование функции создающей обновленный список словарей с информацией о самолетах.
+ Тестирование записи в файл обновленной информации о самолетах.

### Для модуля read_products_json.py:

+ Тестирование создания объектов класса `Category` и `Product` из файла при валидных значениях.
+ Тестирование работы функции при несуществующем пути до файла.

### Для модуля utils.py:

+ Тестирование фильтрации самолетов по высоте полета, заданной пользователем при корректном вводе данных.
+ Тестирование фильтрации самолетов по высоте полета, заданной пользователем при некорректном вводе данных.
+ Тестирование сортировки и возврата топ количества самолетов по высоте полета, заданной пользователем 
при корректном вводе данных.
+ Тестирование сортировки и возврата топ количества самолетов по высоте полета, заданной пользователем 
при некорректном вводе данных.
+ Тестирование фильтрации списка самолетов по стране регистрации при корректном вводе данных.
