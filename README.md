# Проект my_course_planes

## Описание:

Проект my_course_planes - это тестовое приложение для получения информации о самолетах на территории страны.

## Установка:

1. Клонируйте репозиторий:
    ```
    https://github.com/ViktorPilot/E_commerce.git
    ```
2. Запустите PyCharm у себя на компьютере.
3. Откройте скачанный репозиторий в PyCharm.
4. Создайте виртуальное окружение командой `poetry install`.
5. Активируйте ВО командой `poetry env activate`.
6. Установите зависимости командой `poetry init`.
7. Программа готова к работе.

## Запуск проекта через главный модуль main.py:

1. В Python файле `main.py` задать:
    + экземпляры класса 'Product' в формате: ```product1 = Product("Samsung Galaxy S23 Ultra", "256GB, 
         Серый цвет, 200MP камера", 180000.0, 5)```.
    + экземпляры класса 'Category' в формате: ```category1 = Category("Смартфоны",
        "Смартфоны, как средство не только коммуникации, но и получения дополнительных функций для удобства жизни",
        [product1, product2, product3])```.
    + экземпляры класса 'Smartphone' в формате: ```smartphone1 = Smartphone("Samsung Galaxy S23 Ultra", "256GB, 
      Серый цвет, 200MP камера", 180000.0, 5, 95.5, "S23 Ultra", 256, "Серый")```.
    + экземпляры класса 'LawnGrass' в формате: ```grass1 = LawnGrass("Газонная трава", "Элитная трава для газона", 
      500.0, 20, "Россия", "7 дней", "Зеленый")```.
2. Далее необходимо запустить выполнение программы командой `poetry run python main.py`.

## Использование отдельных модулей:

1. В Python файле `category.py` задать:
    + экземпляры класса `Category` в формате: ```category1 = Category("Смартфоны",

        [product1, product2, product3],

2. Запустить выполнение программы командой `python -m src.category`.
3. В результате получим экземпляры класса `Category`.

### Для модуля product.py:

1. В Python файле `product.py` задать:
    + экземпляры класса `Product` в формате: ```product1 = Product("Samsung Galaxy S23 Ultra", "256GB,

2. Запустить выполнение программы командой `python -m src.order`.
3. В результате получим экземпляры класса `OrderProduct`.

1. В Python файле `read_products_json.py` необходимо ввести путь до json-файла с категориями продуктов

2. Запустить выполнение программы командой `python -m src.read_products_json`.
3. В результате получим список экземпляров класса из json-файла.

### Модуль base.py:

1. Файл `base.py` содержит родительский абстрактный класс `BaseProduct` для классов `Smartphone` и `LawnGrass`.
   Он позволяет задать магические методы __init__ и __add__ для дочерних классов.

### Модуль base_cat_and_ord.py:

1. Файл `base_cat_and_ord.py` содержит родительский абстрактный класс `BaseCatOrd` для классов `Category`
   и `OrderProduct`. Он позволяет задать магические методы __init__ и __str__ для дочерних классов.

### Модуль mixin_prod.py:

1. Файл `mixin_prod.py` является миксином для класса `Product`.
   Он позволяет выводить в консоль основную информации о товаре при инициализации.

## Тестирование:

Данный проект покрыт юнит-тестами с использованием фреймворка pytest.
Для их запуска выполните команду `poetry run pytest` из корневой директории проекта.

### Реализованы:

### Для модуля category.py:

+ Тестирование создания экземпляров класса `Category`, работы счетчика категорий и счетчика товаров
  при валидных значениях.
+ Тестирование метода класса `Category`, позволяющего добавлять товар в категорию, если товары из одного класса.
+ Тестирование метода класса `Category`, позволяющего добавлять товар в категорию, если добавляемый товар из
  дочернего класса.
+ Тестирование метода класса `Category`, позволяющего добавлять товар в категорию, если добавляемый товар
  не принадлежит этому и дочерним классам.
+ Тестирование геттера, возвращающего строку информации о товарах заданной категории.
+ Тестирование магического метода __str__, возвращающего строку с названием категории и общем количестве товаров в ней.

### Для модуля product.py:

+ Тестирование создания экземпляра класса `Product` при валидных значениях.
+ Тестирование классметода, позволяющего создать объект класса 'Product', суммирущего общее количество
  и выбирающего максимальную цену товара при наличии товаров с тем же названием.
+ Тестирование геттера класса `Product`, позволяющего вызвать цену товара с приватным уровнем доступа.
+ Тестирование сеттера класса `Product`, позволяющего изменять цену товара с приватным уровнем доступа.
+ Тестирование магического метода __str__, возвращающего информацию о товаре.
+ Тестирование магического метода __add__, возвращающего сумму произведений цены на количество у двух товаров.
+ Тестирование возникновения исключения при суммировании товаров из разных категорий.

### Для модуля order.py:

+ Тестирование создания экземпляра класса `OrderProduct` при валидных значениях.
+ Тестирование магического метода __str__, возвращающего строку с названием, количеством и стоимостью товара.

### Для модуля read_products_json.py:

+ Тестирование создания объектов класса `Category` и `Product` из файла при валидных значениях.
+ Тестирование работы функции при несуществующем пути до файла.

### Для модуля iterator_prod.py:

+ Тестирование работы итератора товаров заданной категории:
    1. Проверка начального значения индекса итератора.
    2. Тестирование работы итератора последовательным вызовом товаров.
    3. Тестирование формирования исключения после окончания перебора всех товаров.

### Для модуля prod_smartphone.py:

+ Тестирование инициализации товара категории `Smartphone` при валидных значениях.

### Для модуля prod_grass.py:

+ Тестирование инициализации товара категории `LawnGrass` при валидных значениях.

### Для модуля mixin_prod.py:

+ Тестирование вывода миксином информации о товаре класса `Product` в консоль.
+ Тестирование вывода миксином информации о товаре класса `Smartphone` в консоль.
+ Тестирование вывода миксином информации о товаре класса `LawnGrass` в консоль.
